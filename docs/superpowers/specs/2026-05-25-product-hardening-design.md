# 产品完善五阶段方案

日期：2026-05-25

## 目标

把当前报表设计器从“核心链路可用”推进到“常用打印报表设计能力更完整、更稳定、更容易验收”的状态。五个阶段按顺序推进：

1. 页面外观闭环：页面水印和页面边框。
2. 组件属性矩阵补齐。
3. 表格设计能力增强。
4. Band 核心严测与行为补齐。
5. 事件系统产品化。

所有阶段都遵守同一条产品原则：模板属性必须贯通设计器画布、RenderDocument、预览、打印 HTML、PDF 导出和测试。除纸张尺寸、缩放视图、浏览器打印机驱动差异外，预览、打印、PDF 应表现一致。

## 非目标

- 不做图表打印。
- 不增加非 JSON 类型数据源。
- 不引入双套未发布规则，也不在代码、UI、文档中使用外部产品名称做能力描述。
- 不做大型视觉换皮；只在涉及的面板、弹窗、右键菜单和入口上继续收敛布局。
- 不在这个阶段重写分页引擎；Band 阶段以合同测试驱动补齐缺口。

## 阶段一：页面外观闭环

### 能力范围

页面继续作为页面级属性中心。当前已经有页面名称和背景色，本阶段新增：

- 页面文字水印。
- 页面边框。

水印第一版只支持文字，不支持图片水印。页面边框支持四边开关、样式、线宽、颜色和距离页面边缘的内偏移。两者都要出现在页面属性区和页面设置弹窗中。

### 模型

在页面模板上增加：

```ts
export interface PageWatermark {
  enabled: boolean;
  text: string;
  fontFamily?: string;
  fontSize: number;
  color: string;
  opacity: number;
  angle: number;
  horizontalAlign: 'left' | 'center' | 'right';
  verticalAlign: 'top' | 'middle' | 'bottom';
  showBehind: boolean;
}

export interface PageBorder {
  enabled: boolean;
  style: 'solid' | 'dashed' | 'dotted' | 'double';
  width: number;
  color: string;
  sides: {
    top: boolean;
    right: boolean;
    bottom: boolean;
    left: boolean;
  };
  offset: number;
}
```

默认值：

- 水印禁用，文本为空，字号 48，颜色 `#8c8c8c`，透明度 `0.18`，角度 `-35`，水平居中、垂直居中，默认在内容后方。
- 边框禁用，实线，线宽 `0.2mm`，颜色 `#000000`，四边开启，偏移 `0mm`。

`RenderPage` 要带同名配置。分页时从模板页复制到渲染页，不参与 Band 高度和分页计算。

### 渲染语义

- 页面背景最底层。
- `showBehind=true` 的水印在 Band 和组件之后的视觉层级下方，但在页面背景之上。
- 页面边框不影响可打印区域、不影响 Band 坐标、不参与碰撞计算。
- `showBehind=false` 的水印在 Band 和组件上方显示，用于盖章类效果。
- 设计器画布、DOM 预览、打印 HTML、PDF 导出都使用相同毫米坐标和相同默认值。

### UI

未选中组件或 Band 时，右侧页面属性显示页面外观分组。页面设置弹窗增加“页面外观”区域，和右侧属性使用同一套字段：

- 水印启用。
- 水印文本。
- 字体、字号、颜色、透明度、角度。
- 水平和垂直对齐，使用图标按钮。
- 显示在内容后方。
- 页面边框启用。
- 边框样式、宽度、颜色、四边、偏移。

所有文案支持中文和英文。属性面板中单位跟随报表单位设置，字段名称不再重复写 `(mm)`。

### 验收

- 新建模板的水印和边框默认禁用。
- 打开示例后可在页面属性区开启水印和边框。
- 设计器画布、预览、打印 HTML、PDF 导出都有相同水印、边框、背景色层级。
- 缩放画布时水印和边框跟随纸张缩放，标尺和可打印区域不被影响。
- 页面设置弹窗应用后，页面属性、画布、预览同步变化。

## 阶段二：组件属性矩阵补齐

### 能力范围

为每类常用组件建立属性矩阵，并按“已有渲染但属性入口缺失”优先补齐。组件范围：

- 文本。
- 图片。
- 表格。
- 富文本。
- 条码。
- 复选框。
- 线条。
- 形状。
- 面板。
- 子报表。
- 页码。
- 日期时间。

### 属性分组

属性面板使用稳定分组：

- 常规：名称、显示名称、类型只读信息。
- 位置：X、Y、宽、高、层级。
- 内容：组件自身内容表达式，例如文本内容、图片地址、条码值、复选框选中表达式、富文本内容、子报表模板。
- 外观：背景、边框、填充、透明度、形状类型、图片适配。
- 文本：字体、字号、加粗、斜体、下划线、删除线、颜色、水平对齐、垂直对齐、格式化。
- 行为：可见、可打印、启用、可增长、可收缩、可拆分。
- 数据：只对表格、子报表等真正需要集合绑定的组件显示。
- 事件：组件事件入口。

“文本内容”是表达式入口，既可以输入普通字符串，也可以输入表达式。文本组件不再同时暴露数据源、字段和表达式三套入口。其他组件也按“一个主要表达式入口”组织。

### 优先补齐项

- 线条：颜色、线宽、线型、起止方向。
- 形状：形状类型、填充、边框颜色、边框宽度、边框样式。
- 图片：适配模式、透明度、边框、可打印表达式。
- 条码：格式、值表达式、显示文本、边距、前景色、背景色。
- 复选框：选中表达式、标签、框样式、勾选样式、文本样式。
- 富文本：双击编辑、字体注册、基础段落工具、预览/打印/PDF 输出一致性。
- 面板：背景、边框、内边距、子组件容器行为。
- 子报表：模板地址、参数表达式、缺失模板占位、最大嵌套限制提示。
- 页码：页码格式、字体、对齐。
- 日期时间：日期时间格式、字体、对齐。

### 验收

- 每类组件有一份属性矩阵文档和对应 UI 覆盖。
- 属性面板不会显示与当前组件无关的控件。
- 每个新增属性至少覆盖画布或预览中一个可见验收点。
- 涉及渲染的属性都覆盖 DOM 预览、打印 HTML、PDF 导出中的一致性测试。

## 阶段三：表格设计能力增强

### 能力范围

表格是高频打印控件。本阶段聚焦设计操作、单元格属性和重复打印：

- 单元格属性：文本、表达式、背景、边框、内边距、对齐、格式化。
- 右键菜单：插入行、删除行、插入列、删除列、合并、拆分、清空内容、设置表头、设置表尾。
- 行列操作：保持单元格坐标、跨度、宽高和现有内容稳定。
- 合并拆分：支持矩形区域合并，拆分后恢复单元格占位。
- 表头重复打印：跨页时重复 header rows。
- 表格可拆分：跨页时按行拆分，单行高于可打印区域时标记溢出。

### 数据与渲染语义

- 表格绑定仍只支持 JSON 数据源。
- 单元格文本是表达式入口，可引用当前行字段。
- 表头行不绑定数据行，跨页重复时只复制表头视觉内容。
- 表尾行可以放合计表达式，仍按现有聚合规则解析。
- 单元格边框和背景在 DOM、打印 HTML、PDF 中保持一致。

### UI

- 表格选中时显示表格整体属性。
- 单元格选中时显示单元格属性，支持多选后批量设置边框、背景、对齐。
- 右键菜单使用紧凑分组，危险操作放在底部。
- 工具按钮使用图标优先，必要时配短文本。

### 验收

- 可以通过右键插入/删除行列。
- 可以合并一个矩形区域并拆分回普通单元格。
- 可以设置单元格背景、边框、文本表达式、对齐和格式化。
- 表格跨页时表头重复打印。
- 预览、打印 HTML、PDF 输出的单元格尺寸、边框和内容一致。

## 阶段四：Band 核心严测

### 能力范围

Band 是分页渲染核心。本阶段不先扩 UI，而是通过合同测试确认行为，再补缺口。测试范围：

- Header/Data/Footer 所属关系。
- 嵌套分组。
- EmptyData。
- KeepTogether。
- CanBreak。
- PrintAtBottom。
- 跨页重复。
- 分组页眉页脚与 DataBand 的顺序。
- 页合计、组合同计、报表合计在分页后的值。

### 行为规则

- DataBand 绑定表后按排序、过滤、分组规则生成逻辑行，再进入分页。
- HeaderBand 在所属 DataBand 或组前打印，可配置跨页重复。
- FooterBand 在所属数据或组后打印，可用于合计。
- EmptyDataBand 在数据源无记录时打印，且不触发普通 DataBand 明细。
- KeepTogether 优先尝试整体移到下一页；仍放不下时按 CanBreak 决定拆分或溢出。
- PrintAtBottom 在当前页底部打印，不推进普通内容流到页脚区域。

### 验收

- 每条规则都有核心合同测试。
- 复杂模板分页结果稳定，页数、Band 顺序、Y 坐标和合计值可断言。
- 修改 Band 行为时失败测试能准确定位到具体规则。

## 阶段五：事件系统产品化

### 能力范围

事件运行时和代码编辑器已有基础，本阶段补齐产品化入口和排错能力：

- 事件入口统一：报表、页面、Band、组件都用同一事件按钮和同一编辑窗体。
- 事件日志面板：显示执行时间、对象、事件名、等级、消息、错误位置。
- 脚本模板：打印前、预览前、渲染前、数据前、组件渲染前、Band 渲染前等常用模板。
- 错误定位：脚本执行错误能回到编辑器，并定位行列。
- 类型提示：继续使用已定义的 `ctx` 类型，模板和自动补全都围绕这个上下文。
- 动态能力：事件内可以改组件属性、隐藏组件、改表达式、创建简单组件。

### 安全边界

- 事件脚本在受控上下文执行，不暴露浏览器全局对象。
- 执行超时、运行错误、语法错误都写入事件日志，不中断整个应用。
- 动态创建组件必须经过模板模型校验。
- 打印和预览使用不同 mode，但事件 API 一致。

### 验收

- 报表、页面、Band、组件都有事件入口。
- 可以插入脚本模板并保存。
- 预览时事件日志能显示成功日志和错误日志。
- 脚本错误能定位到编辑器行列。
- 事件可动态修改文本内容、可见性、样式和简单组件。

## 横向要求

### 国际化

新增 UI 文案必须同步中文和英文。切换语言后，左侧面板、页面设置、属性面板、弹窗、右键菜单、日志面板都应同步切换。

### 测试

每个阶段至少包含：

- 核心模型或渲染合同测试。
- 设计器属性或交互测试。
- 预览/打印/PDF 一致性测试。
- 必要的浏览器烟测。

最终验证命令：

```bash
pnpm test
pnpm build
```

### 提交

每个阶段独立提交，提交信息使用中文 scope：

```text
feat(page): 支持页面水印和边框
feat(properties): 补齐组件属性矩阵
feat(table): 增强表格设计操作
test(band): 补充Band分页合同测试
feat(events): 完善事件日志和模板
```
