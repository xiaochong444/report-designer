# 图表组件设计

**日期**：2026-05-28  
**目标**：为报表设计器增加基础图表能力，覆盖点/线/柱/面积/饼及常见变种，并保证预览、打印、PDF 输出一致。数据源仅支持 JSON。

## 背景

当前报表系统已具备文本、表格、条码、富文本、分页和导出链路，但缺少可配置的图表组件。用户希望图表像常规报表组件一样被拖入 Band，能通过属性面板完成绑定、样式和类型配置，并且在预览、打印、PDF 中保持一致。

## 设计目标

- 支持基础图表类型：散点、折线、柱状、面积、饼图
- 支持常见变种：平滑线、阶梯线、分组柱、堆叠柱、横向柱、堆叠面积、环形饼
- 支持 JSON 数据源绑定和字段映射
- 支持图例、标题、坐标轴、标签、颜色、空值与格式化
- 支持打印和 PDF 导出，并尽量保持与预览一致
- 设计器内提供可编辑属性面板和组件面板入口

## 非目标

- 不做图表交互分析、钻取、缩放、联动筛选
- 不做非 JSON 数据源
- 不做复杂动画与仪表盘级交互

## 方案选择

### 方案 A：完全自研 SVG 图表渲染

优点：
- PDF、打印和预览一致性最好
- 依赖少，输出可控

缺点：
- 开发量大，后续维护成本高
- 轴、标签、堆叠、饼图切分等边界复杂

### 方案 B：接入 `@visactor/react-vchart` 作为图表渲染核心

优点：
- 图表能力完整，基础图表和常见变种覆盖好
- 配置式 spec 与设计器属性模型天然适配
- 支持导出图片，便于打印和 PDF

缺点：
- 运行时依赖更重
- PDF 最稳妥方案是图片快照，矢量导出后续再补

### 方案 C：接入轻量 SVG 库并自行补齐报表层适配

优点：
- 比完全自研快

缺点：
- 仍然需要补很多报表专用行为
- 选择空间不如 VChart 完整

**结论**：采用方案 B。图表的可视化由 VChart 负责，报表系统负责数据绑定、属性配置、预览打印一致性和导出快照。

## 组件模型

新增 `chart` 组件类型，作为普通报表组件插入 Band。

核心属性分三层：

### 1. 数据绑定

- `dataSourceId`：绑定 JSON 数据源
- `categoryField`：类目字段
- `valueFields`：数值字段列表
- `seriesField`：系列字段，可选
- `labelField`：标签字段，可选
- `filterExpression`：过滤表达式，可选
- `sort`：排序字段与方向，可选
- `aggregate`：是否聚合及聚合方式

### 2. 图表行为

- `chartType`：`point | line | bar | area | pie`
- `variant`：
  - point：`scatter`
  - line：`default | smooth | step`
  - bar：`vertical | horizontal | grouped | stacked`
  - area：`default | smooth | stacked`
  - pie：`pie | donut`
- `stackMode`：`none | normal`
- `showLegend`
- `showAxes`
- `showGrid`
- `showLabels`
- `showTooltips`：设计器可保留，打印不依赖
- `palette`

### 3. 外观

- 标题
- 图例位置
- 坐标轴标题
- 字体
- 字号
- 颜色
- 圆角
- 留白
- 空值显示
- 百分比/数值格式

## 渲染链路

### 设计器

图表组件从左侧组件面板拖入画布，默认生成名称。属性面板分为：

- 数据绑定
- 图表类型
- 图例和坐标轴
- 标签和格式
- 外观

### 预览

预览直接使用 VChart 渲染图表。图表数据由 JSON 行和表达式解析结果转换后传入 spec。

### 打印

打印阶段优先保证内容一致。图表在打印前生成稳定快照，输出为与预览尺寸一致的图片或等效静态内容。

### PDF

PDF 导出复用同一份图表快照，避免浏览器和 PDF 之间的排版偏差。

## 数据流

1. 模板保存图表配置
2. 核心层解析 JSON 行数据
3. 图表绑定层生成图表数据集
4. 生成 VChart spec
5. 预览渲染或快照导出
6. 打印/PDF 使用同一份结果

## 校验与错误处理

- 数据源缺失时显示占位提示
- 字段不存在时显示空图状态而不是崩溃
- 数值字段无法解析时按空值处理
- 图表配置缺项时使用默认值
- 导出失败时降级为占位框并记录错误

## 测试策略

- core：组件模型、默认值、绑定解析、排序、聚合、渲染文档结构
- viewer：DOM 预览、打印 HTML、PDF 导出快照一致性
- designer：拖拽入口、属性面板、国际化、示例模板
- 端到端：至少覆盖一种折线图、柱状图、饼图的预览与导出
