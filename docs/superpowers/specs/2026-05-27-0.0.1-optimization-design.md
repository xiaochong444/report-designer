# 0.0.1 Release Optimization Design

**Goal:** 在不扩大量级新功能的前提下，完成一轮发布前优化，重点提升顶部 ribbon 的分区清晰度、多选排版入口、紧凑度和整体可用性。

**Architecture:** 保留现有设计器主结构，只重排 ribbon 职责边界，不改核心模板模型和分页引擎。将常用操作留在 Home，将插入能力留在 Insert，将页面设置留在 Page Layout，将多选组件排版能力集中到 Layout，减少 Home 的拥挤感并提高入口可发现性。

**Tech Stack:** React, TypeScript, Ant Design 6, Vitest, Testing Library, CSS

---

## 范围

本轮只做发布前优化，不扩展大功能面。

### 1. Ribbon 重新分区

- Home：文件、历史、剪贴板、字体、文本对齐、样式入口。
- Insert：数据源、Band 向导、组件插入。
- Page Layout：纸张、页边距、页面设置。
- Layout：多选组件排版。
- Preview：预览入口。

### 2. 多选排版入口完整保留

Layout 中必须保留：

- 左对齐、水平居中、右对齐。
- 上对齐、垂直居中、下对齐。
- 水平平均分布、垂直平均分布。
- 等宽、等高、等尺寸。
- 置于顶层、置于底层。

### 3. 顶部视觉优化

- ribbon 内容区不再靠换行撑高。
- 工具组优先横向滚动，避免把按钮挤成多行。
- tab 与 group 的视觉层级更清晰。
- 图标优先，减少文字占位。

### 4. 国际化

- 新增 ribbon 布局相关文案的中文和英文。
- 切换语言后 tab、按钮、分组标题同步变化。

## 非目标

- 不做图表打印。
- 不重写分页引擎。
- 不新增新的数据源类型。
- 不做大规模视觉换肤。

## 验收

- ribbon 顶部出现 Layout 页签。
- Home 不再承载多选排版操作。
- Layout 中可以完成多选组件的对齐、分布和尺寸调整。
- ribbon 在内容较多时保持单行布局并允许横向滚动。
- 中英文都能正确显示新增 tab 和分组标题。

## 测试

- ribbon tab 切换测试。
- 生产力/多选排版按钮测试。
- 浏览器预览检查 ribbon 高度和分区清晰度。
