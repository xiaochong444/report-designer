## 2. 项目结构

```text
packages/
  core/       模板模型、表达式、Band 计划、分页、布局、渲染文档、事件运行时
  designer/   设计器 UI、画布、属性面板、左侧面板、样式库、条件格式、事件编辑器
  viewer/     预览、打印 iframe、PDF 导出、DOM 渲染器
  example/    示例应用与示例模板
docs/
  superpowers/specs/  已落地方案
  superpowers/plans/  阶段实施计划
```

核心原则：

- 数据源只支持 JSON。
- 图表打印暂不做。
- 设计器交互和界面风格尽量靠近主流打印报表设计器。
- 预览、打印、PDF 应尽量由同一个 `RenderDocument` 输出链路驱动，避免显示不一致。
- 新功能尽量先补测试，再实现，再跑全量验证。

## 3. 关键模块说明

### 3.1 模板与渲染核心

主要文件：

- `packages/core/src/template-model/types.ts`
- `packages/core/src/template-model/template.ts`
- `packages/core/src/pagination/paginate.ts`
- `packages/core/src/render-document/types.ts`
- `packages/core/src/layout-engine/layout-band.ts`

当前模型重点：

- `Page` 已支持 `name`、`backgroundColor`、纸张尺寸、方向、边距、Band 列表。
- `RenderPage` 已带 `backgroundColor`，用于 DOM 预览、打印 HTML、PDF 导出保持一致。
- Band 渲染是核心，已有 Header/Data/Footer、Group、ReportTitle、PageHeader、PageFooter 等常用 Band 能力。
- DataBand 支持绑定 JSON 表数据，渲染前可以按排序规则排序。
- 常用表达式、聚合、页合计、报表合计已经有基础测试覆盖。

### 3.2 设计器

主要文件：

- `packages/designer/src/components/Designer.tsx`
- `packages/designer/src/components/Canvas.tsx`
- `packages/designer/src/components/panels/DesignerPropertyPanel.tsx`
- `packages/designer/src/components/PropertyEditor.tsx`
- `packages/designer/src/components/LeftPanel.tsx`
- `packages/designer/src/components/dialogs/PageSetupDialog.tsx`
- `packages/designer/src/store/designer-store.ts`
- `packages/designer/src/i18n/messages.ts`

当前能力：

- Ribbon 页签、左侧组件/报表树/数据源面板、设计画布、右侧属性区已经成型。
- 画布支持标尺、纸张、页边距、Band 标题条、缩放对齐。
- 页面属性在未选中组件/Band 时展示。
- 页面设置弹窗支持纸张、单位、方向、宽高、边距、页面名称、背景色。
- 左侧报表树显示自动生成或用户编辑的名称，组件拖入后会自动命名。
- 数据源树和表达式编辑器已改为更像报表设计器的树形结构。
- 中文/英文双语基本贯通，新增 UI 文案必须同步更新 `messages.ts`。

### 3.3 Viewer、打印与 PDF

主要文件：

- `packages/viewer/src/components/Viewer.tsx`
- `packages/viewer/src/renderers/dom/RenderDocumentView.tsx`
- `packages/viewer/src/print/print-frame.ts`
- `packages/viewer/src/export/pdf/export-render-document.ts`
- `packages/viewer/src/export/pdf/pdf-draw-component.ts`

当前能力：

- Viewer 支持分页预览、工具栏翻页、切页时自动滚动到当前页。
- Viewer 工具栏在有事件日志时显示日志入口，右侧日志面板展示级别、对象、事件、消息、时间戳和脚本行列，并可触发宿主回到设计器定位对应事件。
- DOM 预览和打印 HTML 共用 `RenderDocument`。
- 打印 HTML 已支持页面背景色。
- PDF 导出已先绘制页面背景色，再绘制组件，避免 PDF 白底和预览不一致。
- 字体注册样式已注入预览/打印链路。

## 6. 已知注意点

1. `packages/example/dist/index.html` 会在构建时更新 JS hash。除非明确要提交构建产物，否则只因构建产生的 hash 改动应恢复。

2. 5184 示例服务可能已在后台运行。启动前可检查端口：

```powershell
Get-NetTCPConnection -LocalPort 5184 -State Listen -ErrorAction SilentlyContinue
```

3. 浏览器插件在某些弹窗输入场景可能出现虚拟剪贴板问题。遇到这种情况可以用本地 Playwright 脚本验证，不要误判为应用 bug。

4. 属性面板已大量依赖 Ant Design 6。新增 UI 时不要使用过期属性。

5. 新增属性必须考虑：

- 模板类型
- store 更新
- 设计器属性面板
- 画布表现
- RenderDocument 输出
- DOM 预览
- 打印 HTML
- PDF 导出
- 中文/英文文案
- 测试覆盖

## 7. 下一步建议

建议优先级如下：

1. 继续补页面属性  
   可补：水印、页面边框、列设置、页面事件入口、页面启用/可打印表达式。

2. 继续补组件属性  
   文本、图片、表格、富文本、条码、复选框、线条、形状、面板、子报表、页码、日期时间都应逐项对齐“属性面板 -> 画布 -> RenderDocument -> 预览/打印/PDF”。

3. 表格能力继续增强  
   重点是右键菜单、单元格属性、行列插入删除、合并拆分、边框、单元格表达式、表头/表尾重复、数据绑定。

4. Band 能力继续稳固  
   重点是 DataBand 分页、分组页眉页脚、KeepTogether、CanBreak、PrintOn、空数据打印、页合计/组合计/报表合计。

5. 事件系统继续产品化  
   已有 Monaco、类型上下文、脚本模板、运行日志面板、运行时错误定位，以及从日志定位回设计器事件编辑器。下一轮建议补事件日志筛选、日志导出、更多事件钩子。

6. 富文本继续打磨  
   当前使用 Tiptap。后续重点是双击编辑稳定性、字体注册、字号/颜色/加粗等工具栏、HTML 清洗、预览/打印/PDF 输出一致性。

7. UI 继续收敛  
   左侧面板、属性区、样式库、条件格式窗体还可以继续压缩布局、图标化、统一间距和滚动行为。

## 8. 开发约定

- 先写失败测试，再实现。
- 改 UI 时同步中英文。
- 改渲染时至少覆盖 DOM 预览和打印，涉及导出时也要覆盖 PDF。
- 不保留未发布版本的双套规则，不在代码和 UI 中出现内部版本标记。
- 避免引入其他产品名称到 UI、文档或代码。
- 提交信息使用中文并带 scope，例如：

```text
feat(page): 支持页面背景色渲染
fix(viewer): 修复预览翻页滚动
test(render): 补充打印一致性用例
```
