# ERP 单据打印框架 — 选型功能清单

> 本文档列出专业级 Web 打印框架应具备的功能维度，作为选型评估的 checklist。
> 后续对照此清单逐项评估候选产品。

---

## 一、模板设计能力

| # | 功能项 | 说明 | 优先级 |
|---|--------|------|:------:|
| 1.1 | 可视化设计器 | 拖拽式画布设计打印模板，所见即所得 | 必选 |
| 1.2 | 元素类型 | 支持文本、图片、线条、矩形、条码、二维码、表格、富文本 | 必选 |
| 1.3 | 表格支持 | 支持动态行表格，能处理跨页自动截断、表头重复、行高自适应 | 必选 |
| 1.4 | 数据绑定 | 模板字段与 JSON 数据通过键名绑定，运行时自动填充 | 必选 |
| 1.5 | 表达式/计算 | 支持字段间运算（如金额合计 = Σ 单价 × 数量），条件渲染 | 必选 |
| 1.6 | 联次管理 | 同一单据支持多联（供应商联/仓库联/财务联/存根联），各联内容/水印可差异化 | 推荐 |
| 1.7 | 模板版本 | 模板支持版本管理，历史单据打印不受模板变更影响 | 推荐 |
| 1.8 | 模板导入导出 | 模板可导出为 JSON/XML 文件存储，支持导入复用 | 推荐 |
| 1.9 | 栅格/辅助线 | 设计器提供辅助线、标尺、网格对齐，辅助精确定位 | 推荐 |
| 1.10 | 模板预览 | 设计器内可直接预览填充效果 | 推荐 |

## 二、打印输出能力

| # | 功能项 | 说明 | 优先级 |
|---|--------|------|:------:|
| 2.1 | 浏览器打印 | 调用 `window.print()`，通过浏览器打印对话框输出 | 必选 |
| 2.2 | 静默打印 | 无需弹窗直接发送到打印机（需客户端插件或特殊权限） | 推荐 |
| 2.3 | 打印预览 | 独立预览窗口/弹窗，所见即所打 | 必选 |
| 2.4 | 分页控制 | 支持自动分页、强制分页（`page-break-before/after`）、防止表格行截断 | 必选 |
| 2.5 | 页眉/页脚 | 每页自动重复页眉页脚，支持"第 N 页 / 共 M 页" | 必选 |
| 2.6 | 精确尺寸 | 支持 mm/pt 单位精确定位，满足针式打印多联单据对齐需求 | 必选 |
| 2.7 | 纸张设置 | 支持 A4、A5、B5、自定义纸张尺寸，连续纸设置 | 必选 |
| 2.8 | 横向/纵向 | 支持横版（Landscape）和竖版（Portrait）打印 | 必选 |
| 2.9 | 批量打印 | 支持一次性提交多份单据到打印队列 | 推荐 |
| 2.10 | PDF 导出 | 支持将打印内容导出为 PDF 文件下载 | 推荐 |
| 2.11 | 多份打印 | 支持指定打印份数、逐份打印 | 推荐 |
| 2.12 | 指定打印机 | 能选择目标打印机（浏览器内可选或插件级指定） | 推荐 |
| 2.13 | 边距微调 | 支持上下左右打印偏移补偿（±mm），修正针式打印走纸偏差 | 必选 |

## 四、ERP 单据专属能力

| # | 功能项 | 说明 | 优先级 |
|---|--------|------|:------:|
| 4.1 | 表格合计行 | 自动在表格尾部生成合计/汇总行（总金额、总数量等） | 必选 |
| 4.2 | 金额大写 | 金额字段自动转换为中文大写（零壹贰叁肆伍陆柒捌玖拾佰仟万亿圆整） | 必选 |
| 4.3 | 文本溢出/换行 | 单元格长文本自动换行、截断处理，不破坏打印布局 | 必选 |
| 4.4 | 条件显示/隐藏 | 根据数据内容动态显示或隐藏指定区域（如折扣为 0 时隐藏折扣行） | 必选 |
| 4.5 | 公司信息自动填充 | 打印头部公司名称、电话、地址、税号等从系统配置自动读取 | 推荐 |
| 4.6 | 打印偏移记忆 | 按打印机 + 纸张类型记忆偏移补偿值，一次校准长期使用 | 推荐 |
| 4.7 | 多语言模板 | 同一模板支持中英文等多语言切换 | 可选 |
| 4.8 | 单据编号水印 | 打印时自动叠加单据编号作为防伪水印 | 可选 |

## 十一、当前 SCM 系统的重点关注项

结合本项目实际情况（SCM 供应链管理、Taro/React 技术栈、钉钉环境）：

| 关注项 | 原因 |
|--------|------|
| 钉钉浏览器兼容 | 系统部署在钉钉微应用中 |
| 针式打印支持 | 采购单/入库单可能需要多联打印 |
| 模板可视化设计 | 业务人员需自行调整单据模板 |
| React 集成 | 前端基于 React 开发 |
| TypeScript 类型 | 项目使用 TypeScript |
| 条码生成 | 单据上需要打印条码（款号、单号） |
| 表格跨页 | 单据明细行数不确定，需处理跨页 |
| 纯前端 or 轻量插件 | 避免给每个终端用户安装重型客户端 |
